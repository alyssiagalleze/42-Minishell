- echo (-n)
	displays arg in stdout
	'\' and -e doesn't have to be implemented, so no escape sequence is supposed to work
	
	
	test cases :
		echo with no arg		-> displays a newline
		echo hello\nworld		-> hellonworld
		echo "hello\nworld"		-> hello\nworld

	-n : removes the final newline

	exit status	: 0 unless a write error occurs

- cd (change directory)
	cd	|| cd ~			-> moves to home folder (~ = HOME = /home/agalleze)
	cd + tab			-> displays all available directories and files, directories suffixed with /
	cd .				-> move to working directory (no change)
	cd ..				-> moves to parent directory
	cd -				-> access the last working directory (saved in the OLDPWD env variable)
	cd /<directory>		-> access specified directory (/ prefix is important if directory is absolute)
	cd directory		-> access a directory available in the working directory

					usefull functions, chdir (see authorized functions document)

- pwd (print working directory)
	pwd							->print env variable PWD
	pwd inside a symlink		-> displays the symbolic path
	/bin/pwd					-> displays the "physical" path

		pwd != echo $pwd --> pwd is a program, it does not use the env variable PWD 
							 works even if PWD is unset

- export
	defines environment variables or makes them available to child processes (scripts and programs)

	export w/out args		-> displays all exported variables
	export <name=value>		-> creates a new environment variable set with "value" available to any child process
								double quotes are added in the env list
									-----if variable already exists, change the value
	export <name>			-> creates a new empty environment variable available to any child process
								------displayed only with the <export> command

	EXIT_STATUS :	not a valid identifier					--> 1	
					syntax error near unexpected token 		--> 2	



- unset 
	unset <name>			-> destroys variable name
	unset 					-> gives the prompt back

- env 
	displays current environment : env <name=value> <command <arg> .. >

	env					-> displays all environment variables
	env <name=value>	-> if name doesn't exist, creates a new env variable set with value
						   otherwise, changes the value of the environment variable name

	exit status :	0		-> env command completed succefully
					1-125	-> an error occured in the env command
					126		-> command exists, but could not be launched
					127		-> command was not found


- exit <n>
	exit the shell, return a status of n to the shell's parent.
		if n is omitted, return the status of the last command executed
